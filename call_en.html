My role

Requirement analysis, ux research, interface design


Project time：2018.12

Transforming call center operation and improve customer experience

We used to have one agent make one call to one customer for one business issues. Customers were annoyed with too many calls with different agent if they have several issues. It is also not efficient.

We started with defining the stakeholders and project scope

It is a system designed from scratch, so it is important to know the stakeholders at the beginning.



The users/ stakeholders and their influnce to the system are：

  • After sale manager- obtain performance data
  • Business operator- define business rules
  • Manager- assigne tasks to team members
  • Staff- solve the cases

We investigated the functionity and system structure

With product managers we discuss requirments and tried different solutions on whiteboard.

The key design challenges

We defined our design principles according to our design goal, user bahavior and system complexity.

  • User adaption- the system should adapt user’s working habit and mental modal
  • Business impact- the system should help to collect important business data
  • Scablity- the system can be enlarged for other business cases in future

Document analysis

Shadowing

Obserse the users in their working context and document data

Key findings




  • If the agent get anwser from customer, he/she will fill the form and the case is closed
  • If customer has picked us the phone call but there is no communication, agent need to call again.

Design of the customer list

When designed task list, we need to know how agent manage tasks and choose which task to do.

From the field study, we have learned that agent piorize tasks with following factors:the severity, appointed or not, processing or not.
