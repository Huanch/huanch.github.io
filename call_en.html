Call center 1.0

Improving the working efficieny of agents and customer satisfaction

My role

Requirement analysis, ux research, interface design


Project time：2018.12

We designed and built a new call center system so that the an agent can work with multiple cases in one call.


In Guazi call center, there were different agent teams work for different business lines. Customers were annoyed with too many calls with different agents if they were invoveled with more than one business. It is also not the most efficent for contacting customers.

We attemped to make a new call center system which aggrigates all cases so that an agent could collect information from a customer for all cases in one call.

Analysis of the stakeholders and their requirement

It is a system designed from scratch, so it is important to know the stakeholders and what they expect from the system at the beginning.



The users/ stakeholders and their influnce to the system are：

  • After sale manager- obtain performance data
  • Business operator- define business rules
  • Manager- assigne tasks to team members
  • Staff- solve the cases

We then defined the system structure.

With product managers we discuss requirments and tried different solutions on whiteboard.

Design for efficieny and adaptability

The design goal in the project is to make the system efficient to use and easy to be adapted. For reaching the goal, we need to solve following design challenges:

  • How might we help agent better manage the tasks- the system should adapt user’s working habit and mental modal
  • How might we make the agent working more smoonthly- the system can be enlarged for other business cases in future
  • How might we make a new agent adapt to job fast- the employee replacement is frequent in call center.

We defined our basic design principle while exploring possiblle solutions:

  • contextual- the system should adapt user’s working habit and mental modal
  • Scable- the system can be enlarged for other business cases in future

A field study to understand deeply how agents work

I took a field study in call center to understand the working context and the user behavior.

I analyzed the spreadsheet and observed how an agent had called and updated information.



Key take away




  • The agent piorize tasks with following factors:the severity, appointed or not, in processing or not.
  • The agent needs to ask whether the person who picks up the phone is the customer. If not, the communication ends.
