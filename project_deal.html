Design of an application for managing promotional information of stores

This design is an approach to solve the problems that there are too many promotional plan that we can't remember so much.

Create a map

I start by mapping the main user journey in order to divide the work into small parts

A primary exploration

I break the system into components and I sketch a few alternatives of the components to generate ideas and understand the design problems

How should the information be structured and how user would access the information?

A store can has several deals

User might search for deal by choosing a store or picking a date

User tasks

Following are tasks user need to accomplish on the interface

  • User can add new entry for new store
  • User can assign the new entry to an store which is existed in the system
  • User can take photos of the deal and upload it
  • User can set the period of the deal

Usability goals

Following are usability constrains are to be consided while design solutions

  • How user create a entry quickly?(user dont want to spend much time to fill a entry)

Specify the use case and storyboard the details

I continue to sketch the interaction on sticky notes and paper for each component.
